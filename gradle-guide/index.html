Gradle 入门教程

Jul 10, 2015 in 加工

什么是 Gradle

Gradle 是一种依赖管理工具，基于 Groovy 语言，面向 Java 应用为主，它抛弃了基于 XML 的各种繁琐配置，取而代之的是一种基于 Groovy 的内部领域特定（DSL）语言。

Gradle makes the impossible possible, the possible easy and the easy elegant.

Gradle 相关的几个文件

/build.gradle

整个项目的 gradle 基础配置文件
主要内容是声明仓库源和 android gradle plugin 版本
这里指定的是 jcenter，一个新的中央远程仓库，兼容 maven 仓库而且性能更优

/app/build.gradle

app 文件夹这个 Module 的 gradle 配置文件，算是整个项目最主要的 gradle 配置文件

需要注意的几点：

  • apply plugin 是最新 gradle 版本的写法，以前的写法是 apply plugin: 'android'
  • buildToolsVersion 需要本地已安装，导入新第三方库时可以手动改为本地已有的版本或下载相应的版本
  • minifyEnabled 也是最新的语法，以前是 runProguard
  • proguardFiles 包含系统默认的混淆文件和项目自定义的混淆文件

/gradle/gradle-wrapper.properties

声明 gradle 目录与下载路径以及当前项目使用的 gradle 版本，默认路径一般不改，导包时注意这里声明的版本

/settings.gradle

全局的项目配置文件，声明一些需要加入 gradle 的 module

使用命令行 Gradle 编译项目

  1. 切换到项目根目录

  • 使用 gradlew -v 查看项目所用的 Gradle 版本

    如果本地没有相应版本的 Gradle 会自动下载

  • 执行 gradlew clean 去下载 Gradle 的一些依赖

  1. 执行 gradlew build 直接编译并生成相应的 apk 文件
  2. 生成的 apk 文件在 /app/build/outputs/apk

    需要设置环境变量 ANDROID_HOME，JAVA_HOME

项目导入 Android Studio

  1. File->Import Project 选择项目目录

  • 再次 下载 Gradle

    bug? AS1.2.2此 bug 依旧存在

  • 导入之后几个需要注意的地方

  1. 每个 Module 下的 build.gradle 里的 buildToolsVersion
  2. /build.gradle 里的 gradle 插件版本
  3. /gradle/wrapper/gralde-wrapper.properties 里 Gradle Wrapper 的版本
  4. 老项目的语法问题

Gradle 常用命令

gradlew 代表 gradle wrapper，意味着项目包装了 gradle，使用 gradlew 可直接替代全局 gralde 命令；gradlew 其实就是一个 shell 脚本文件，目的就是使用 gradle-wrapper.jar 来构建你的项目；使用 gradle wrapper 的好处就是每个项目可以依赖不同版本的 gradle，构建的时候 gradle wrapper 会自动下载所依赖的版本的 gradle

  • gradlew -v 版本号
  • gradlew clean 清除 /app/build 并下载依赖
  • gradlew build 检查依赖并编译打包
  • gradlew assembleDebug 编译并打 Debug 包
  • gradlew assembleRelease 编译并打 Release 包
  • gradlew installRelease Release 模式打包并安装
  • gradlew uninstallRelease 卸载 Release 模式包
